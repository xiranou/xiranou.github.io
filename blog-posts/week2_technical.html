Inline vs Inline-block vs Block

Week-2 Technical Blog 08/14/2014

Every HTML element can be consider as a box, a rectangle box. This is called the 'box model' in CSS. The display porperty in CSS determines how these boxes would behave. The most commonly used are inline, inline-block, and block. Inline Inline is the default property of elements like <span>, <em>, or <img>. They have no line breaks before or after them, and sit between a string of text or other elements. An inline element respects left & right margins and padding, but not top & bottom. It doesn't accept width and height. div{border: 1px solid black; background-color: red; height: 100px; width: 100px; display: inline;} Block Container type HTML elements such as <div> <article> <p> or header elemtns (<h1>, <h2>, etc), have a default display property of 'block'. They force line breaks before and after them, so that they sit right ontop of each other. They respect margins and paddings from all sides and accepts width and height. div{border: 1px solid black; background-color: red; height: 100px; width: 100px; display: block;} Inline-Block Lastly, the inline-block display property. Elements with this display property will not have line breaks before and after them, thus they will sit like an inline element, between other elements or a string of text. However, they will accept width, heights and margins & paddings from all sides. I'd like to think of them as the hybrid of inline & block. div{border: 1px solid black; background-color: red; height: 100px; width: 100px; display: inline-block;} You can use CSS to change the display property of HTML elements. This will help greatly when you want your divs to sit next to each-other, or your img tags to stack ontop of each other.
